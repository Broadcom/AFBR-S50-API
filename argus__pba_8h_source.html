44/*!***************************************************************************
45 * @defgroup argus_pba Pixel Binning Algorithm
46 * @ingroup argus_api
47 *
48 * @brief Pixel Binning Algorithm (PBA) parameter definitions and API functions.
49 *
50 * @details Defines the generic Pixel Binning Algorithm (PBA) setup parameters
51 * and data structure.
52 *
53 * The PBA module contains filter algorithms that determine the
54 * pixels with the best signal quality and extract an 1D distance
55 * information from the filtered pixels by averaging them in a
56 * specified way.
57 *
58 * The Pixel Binning Algorithm is a three-stage filter with a
59 * fallback value:
60 *
61 * -# A fixed pre-filter mask is applied to statically disable
62 * specified pixels.
63 * -# A relative and absolute amplitude filter is applied in the
64 * second stage. The relative filter is determined by a ratio
65 * of the maximum amplitude off all available (i.e. not filtered
66 * in stage 1) pixels. Pixels that have an amplitude below the
67 * relative threshold are dismissed. The same holds true for
68 * the absolute amplitude threshold. All pixel with smaller
69 * amplitude are dismissed.\n
70 * Note that the absolute amplitude threshold is disabled if
71 * the Golden Pixel (see below) is also disabled in order to
72 * prevent invalid filtering for multi-pixel devices.\n
73 * The relative threshold is useful to setup a distance
74 * measurement scenario. All well illuminated pixels are
75 * selected and considered for the final 1D distance. The
76 * absolute threshold is used to dismiss pixels that are below
77 * the noise level. The latter would be considered for the 1D
78 * result if the maximum amplitude is already very low.
79 * -# An absolute minimum distance filter is applied in addition
80 * to the amplitude filter. This removes all pixel that have
81 * a lower distance than the specified threshold. This is used
82 * to remove invalid pixels that can be detected by a physically
83 * not correct negative distance.
84 * -# A distance filter is used to distinguish pixels that target
85 * the actual object from pixels that see the brighter background,
86 * e.g. white walls. Thus, the pixel with the minimum distance
87 * is referenced and all pixel that have a distance between
88 * the minimum and the given minimum distance scope are selected
89 * for the 1D distance result. The minimum distance scope is
90 * determined by an relative (to the current minimum distance)
91 * and an absolute value. The larger scope value is the
92 * relevant one, i.e. the relative distance scope can be used
93 * to heed the increasing noise at larger distances.
94 * -# If all of the above filters fail to determine a single valid
95 * pixel, the Golden Pixel is used as a fallback value. The
96 * Golden Pixel is the pixel that sits right at the focus point
97 * of the optics at large distances.
98 * .
99 *
100 * After filtering is done, there may be more than a single pixel
101 * left to determine the 1D signal. Therefore several averaging
102 * methods are implemented to obtain the best 1D result from many
103 * pixels. See #argus_pba_averaging_mode_t for details.
104 *
105 *
106 * @addtogroup argus_pba
107 * @{
