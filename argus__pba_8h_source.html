44/*!***************************************************************************
45 * @defgroup argus_pba Pixel Binning Algorithm
46 * @ingroup argus_api
47 *
48 * @brief Pixel Binning Algorithm (PBA) parameter definitions and API functions.
49 *
50 * @details Defines the generic pixel binning algorithm (PBA) setup parameters
51 * and data structure.
52 *
53 * The PBA module contains filter algorithms that determine the
54 * pixels with the best signal quality and extract an 1d distance
55 * information from the filtered pixels.
56 *
57 * The pixel filter algorithm is a three-stage filter with a
58 * fallback value:
59 *
60 * -# A fixed pre-filter mask is applied to statically disable
61 * specified pixels.
62 * -# A relative and absolute amplitude filter is applied in the
63 * second stage. The relative filter is determined by a ratio
64 * of the maximum amplitude off all available (i.e. not filtered
65 * in stage 1) pixels. Pixels that have an amplitude below the
66 * relative threshold are dismissed. The same holds true for
67 * the absolute amplitude threshold. All pixel with smaller
68 * amplitude are dismissed.\n
69 * The relative threshold is useful to setup a distance
70 * measurement scenario. All well illuminated pixels are
71 * selected and considered for the final 1d distance. The
72 * absolute threshold is used to dismiss pixels that are below
73 * the noise level. The latter would be considered for the 1d
74 * result if the maximum amplitude is already very low.
75 * -# A distance filter is used to distinguish pixels that target
76 * the actual object from pixels that see the brighter background,
77 * e.g. white walls. Thus, the pixel with the minimum distance
78 * is referenced and all pixel that have a distance between
79 * the minimum and the given minimum distance scope are selected
80 * for the 1d distance result. The minimum distance scope is
81 * determined by an relative (to the current minimum distance)
82 * and an absolute value. The larger scope value is the
83 * relevant one, i.e. the relative distance scope can be used
84 * to heed the increasing noise at larger distances.
85 * -# If all of the above filters fail to determine a single valid
86 * pixel, the golden pixel is used as a fallback value. The
87 * golden pixel is the pixel that sits right at the focus point
88 * of the optics at large distances.
89 * .
90 *
91 * After filtering is done, there may be more than a single pixel
92 * left to determine the 1d signal. Therefore several averaging
93 * methods are implemented to obtain the best 1d result from many
94 * pixels. See #argus_pba_averaging_mode_t for details.
95 *
96 *
97 * @addtogroup argus_pba
98 * @{
99 *****************************************************************************/
