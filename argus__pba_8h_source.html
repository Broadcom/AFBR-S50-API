41 /*!***************************************************************************
42  * @defgroup arguspba Pixel Binning Algorithm
43  * @ingroup argusapi
44  *
45  * @brief Pixel Binning Algorithm (PBA) parameter definitions and API functions.
46  *
47  * @details Defines the generic pixel binning algorithm (PBA) setup parameters
48  * and data structure.
49  *
50  * The PBA module contains filter algorithms that determine the
51  * pixels with the best signal quality and extract an 1d distance
52  * information from the filtered pixels.
53  *
54  * The pixel filter algorithm is a three-stage filter with a
55  * fallback value:
56  *
57  * -# A fixed pre-filter mask is applied to statically disable
58  * specified pixels.
59  * -# A relative and absolute amplitude filter is applied in the
60  * second stage. The relative filter is determined by a ratio
61  * of the maximum amplitude off all available (i.e. not filtered
62  * in stage 1) pixels. Pixels that have an amplitude below the
63  * relative threshold are dismissed. The same holds true for
64  * the absolute amplitude threshold. All pixel with smaller
65  * amplitude are dismissed.\n
66  * The relative threshold is useful to setup a distance
67  * measurement scenario. All well illuminated pixels are
68  * selected and considered for the final 1d distance. The
69  * absolute threshold is used to dismiss pixels that are below
70  * the noise level. The latter would be considered for the 1d
71  * result if the maximum amplitude is already very low.
72  * -# A distance filter is used to distinguish pixels that target
73  * the actual object from pixels that see the brighter background,
74  * e.g. white walls. Thus, the pixel with the minimum distance
75  * is referenced and all pixel that have a distance between
76  * the minimum and the given minimum distance scope are selected
77  * for the 1d distance result. The minimum distance scope is
78  * determined by an relative (to the current minimum distance)
79  * and an absolute value. The larger scope value is the
80  * relevant one, i.e. the relative distance scope can be used
81  * to heed the increasing noise at larger distances.
82  * -# If all of the above filters fail to determine a single valid
83  * pixel, the golden pixel is used as a fallback value. The
84  * golden pixel is the pixel that sits right at the focus point
85  * of the optics at large distances.
86  * .
87  *
88  * After filtering is done, there may be more than a single pixel
89  * left to determine the 1d signal. Therefore several averaging
90  * methods are implemented to obtain the best 1d result from many
91  * pixels. See #argus_pba_averaging_mode_t for details.
92  *
93  *
94  * @addtogroup arguspba
95  * @{
