139/*!***************************************************************************
140 * @brief Creates a new device data handle object to store all internal states.
141 *
142 * @details The function must be called to obtain a new device handle object.
143 * The handle is basically an abstract object in memory that contains
144 * all the internal states and settings of the API module. The handle
145 * is passed to all the API methods in order to address the specified
146 * device. This allows to use the API with more than a single measurement
147 * device.
148 *
149 * The handler is created by calling the memory allocation method from
150 * the standard library: @code void * malloc(size_t size) @endcode
151 * In order to implement an individual memory allocation method,
152 * define and implement the following weakly binded method and return
153 * a pointer to the newly allocated memory. *
154 *
155 * @code void * Argus_Malloc (size_t size) @endcode
156 *
157 * Also see the #Argus_DestroyHandle method for the corresponding
158 * deallocation of the allocated memory.
159 *
160 * @note Although the method is using memory allocated on the heap, it
161 * is eventually no dynamic memory allocation, since the block of
162 * memory is kept all the time and no memory blocks are dynamically
163 * freed and re-allocated. If the usage of heap must be avoided, one
164 * can always implement its own version of the `Argus_Malloc` function
165 * to create the memory elsewhere.
166 *
167 * @return Returns a pointer to the newly allocated device handler object.
168 * Returns a null pointer if the allocation failed!
169 *****************************************************************************/

283/*!***************************************************************************
284 * @brief Starts the timer based measurement cycle asynchronously.
285 *
286 * @details This function starts a timer based measurement cycle asynchronously
287 * in the background. A periodic timer interrupt triggers the measurement
288 * frames on the ASIC and the data readout afterwards.
289 *
290 * When the measurement has finished, a callback (which is passed as
291 * a parameter to the function) is invoked in order to inform the
292 * main thread to call the \link #Argus_EvaluateData data evaluation
293 * method\endlink. This call is mandatory to release the data buffer
294 * for the next measurement and it must not be invoked directly from
295 * the callback since it is currently within an interrupt service
296 * routine. Rather a flag should inform the main thread or task
297 * scheduler to invoke the evaluation as soon as possible in order
298 * to not introduce any unwanted delays to the next measurement frame.
299 *
300 * The next measurement frame will be started as soon as the pre-
301 * conditions are meet. These are:
302 * 1. timer flag set (i.e. a certain time has passed since the last
303 * measurement in order to fulfill eye-safety),
304 * 2. device idle (i.e. no measurement currently ongoing) and
305 * 3. data buffer ready (i.e. the previous data has been evaluated).
306 *
307 * Usually, the device idle and data buffer ready conditions are met
308 * before the timer tick occurs and thus the timer dictates the frame
309 * rate.
310 *
311 * The callback function pointer will be invoked when the measurement
312 * frame has finished successfully or whenever an error, that cannot
313 * be handled internally, occurs.
314 *
315 * The periodic timer interrupts are used to check the measurement status
316 * for timeouts. An error is invoked when a measurement cycle have not
317 * finished within the specified time.
318 *
319 * Use #Argus_StopMeasurementTimer to stop the measurements.
320 *
321 * @note In order to use this function, the periodic interrupt timer module
322 * (see @ref argus_timer) must be implemented!
323 *
324 * @param hnd The API handle; contains all internal states and data.
325 * @param cb Callback function that will be invoked when the measurement
326 * is completed. Its parameters are the \link #status_t status
327 * \endlink and a pointer to the \link #argus_results_t results
328 * \endlink structure. If an error occurred, the status differs
329 * from #STATUS_OK and the second parameter is null.
330 * @return Returns the \link #status_t status\endlink (#STATUS_OK on success).
331 *****************************************************************************/
332status_t Argus_StartMeasurementTimer(argus_hnd_t * hnd, argus_callback_t cb);

347 * @brief Triggers a single measurement frame asynchronously.
348 *
349 * @details This function immediately triggers a single measurement frame
350 * asynchronously if all the pre-conditions are met. Otherwise it
351 * returns with a corresponding status (e.g. #STATUS_BUSY or
352 * #STATUS_ARGUS_POWERLIMIT).
353 *
354 * When the measurement has finished, a callback (which is passed as
355 * a parameter to the function) is invoked in order to inform the
356 * main thread to call the \link #Argus_EvaluateData data evaluation
357 * method\endlink. This call is mandatory to release the data buffer
358 * for the next measurement and it must not be invoked directly from
359 * the callback since it is currently within an interrupt service
360 * routine. Rather a flag should inform the main thread or task
361 * scheduler to invoke the evaluation task.
362 *
363 * The pre-conditions for starting a measurement frame are:
364 * 1. timer flag set (i.e. a certain time has passed since the last
365 * measurement in order to fulfill eye-safety),
366 * 2. device idle (i.e. no measurement currently ongoing) and

369 * The callback function pointer will be invoked when the measurement
370 * frame has finished successfully or whenever an error, that cannot
371 * be handled internally, occurs.
372 *
373 * The successful finishing of the measurement frame is not checked

383 * @return Returns the \link #status_t status\endlink (#STATUS_OK on success).
384 *****************************************************************************/

434/*!***************************************************************************
435 * @brief Evaluate useful information from the raw measurement data.
436 *
437 * @details This function is called with a pointer to the raw results obtained
438 * from the measurement cycle. It evaluates this data and creates
439 * useful information from it. Furthermore, calibration is applied to
440 * the data. Finally, the results are used in order to adapt the device
441 * configuration to the ambient conditions in order to achieve optimal
442 * device performance.
443 *
444 * Therefore, it consists of the following sub-functions:
445 * - Apply pre-calibration: Applies calibration steps before evaluating
446 * the data, i.e. calculations that are to the integration results
447 * directly.
448 * - Evaluate data: Calculates measurement parameters such as range,
449 * amplitude or ambient light intensity, depending on the configurations.
450 * - Apply post-calibration: Applies calibrations after evaluation of
451 * measurement data, i.e. calibrations applied to the calculated
452 * values such as range.
453 * - Dynamic Configuration Adaption: checks if the configuration needs
454 * to be adjusted before the next measurement cycle in order to
455 * achieve optimum performance. Note that the configuration might not
456 * applied directly but before the next measurement starts. This is
457 * due to the fact that the device could be busy measuring already
458 * the next frame and thus no SPI activity is allowed.
459 * .
460 * However, if the device is idle, the configuration will be written
461 * immediately.
462 *
463 * @param hnd The API handle; contains all internal states and data.
464 * @param res A pointer to the results structure that will be populated
465 * with evaluated data.
466 * @param raw The pointer to the raw data that has been obtained by the
467 * measurement finished callback.
468 * @return Returns the \link #status_t status\endlink (#STATUS_OK on success).
469 *****************************************************************************/
470status_t Argus_EvaluateData(argus_hnd_t * hnd, argus_results_t * res, void * raw);

495/*!***************************************************************************
496 * @brief Executes a relative range offset calibration measurement.
497 *
498 * @details This function immediately triggers a relative range offset calibration
499 * measurement sequence. The ordinary measurement activity is suspended
500 * while the calibration is ongoing.
501 *
502 * In order to perform a relative range offset calibration, a flat
503 * calibration target must be setup perpendicular to the sensors
504 * field-of-view.
505 *
506 * \code
507 * AFBR-S50 ToF Sensor
508 * #|
509 * #| |
510 * #|-----+ |
511 * #| Rx | |
512 * Reference #|----++ | Calibration
513 * Plane #| Tx | | Target
514 * #|----+ |
515 * #| |
516 * #| <------- targetRange -----------------> |
517 * \endcode
518 *
519 * There are two options to run the offset calibration: relative and
520 * absolute.
521 * - Relative (#Argus_ExecuteRelativeRangeOffsetCalibrationSequence):
522 * when the absolute distance is not essential or the distance to
523 * the calibration target is not known, the relative method can be
524 * used to compensate the relative pixel range offset w.r.t. the
525 * average range. The absolute or global range offset is not changed.
526 * - Absolute (#Argus_ExecuteAbsoluteRangeOffsetCalibrationSequence):
527 * when the absolute distance is essential and the distance to the
528 * calibration target is known, the absolute method can be used to
529 * calibrate the absolute measured distance. Additionally, the
530 * relative pixel offset w.r.t. the average range is also compensated.
531 * .
532 *
533 * After calibration has finished successfully, the obtained data is
534 * applied immediately and can be read from the API using the
535 * #Argus_GetCalibrationPixelRangeOffsets or
536 * #Argus_GetCalibrationGlobalRangeOffset function.
537 *
538 * @param hnd The API handle; contains all internal states and data.
539 * @param mode The targeted measurement mode.
540 * @return Returns the \link #status_t status\endlink (#STATUS_OK on success).
541 *****************************************************************************/

543 argus_mode_t mode);
